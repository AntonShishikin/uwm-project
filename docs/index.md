# GitHub Pages Starter

Эта страница служит точкой входа для публикации проекта на GitHub Pages и описывает, как выкатить веб-версию UWM.

## Что выкладывается

- Клиентская SPA уже лежит в каталоге `web/`.
- Браузерный интерфейс работает без сторонних зависимостей, поэтому его достаточно раздать статически.

## Быстрый запуск GitHub Pages

1. В корне репозитория создайте ветку/директорию `docs/` (эта страница уже внутри неё).
2. Скопируйте файлы из `web/` в `docs/`:
   ```bash
   rsync -av web/ docs/
   ```
3. Добавьте файл `docs/CNAME`, если нужен кастомный домен.
4. В настройках репозитория GitHub включите Pages и укажите источник `Deploy from a branch` → ветка `main` → папка `/docs`.
5. После публикации откройте выданный URL, чтобы убедиться, что SPA загружается.

## Обновление версии

- При изменении исходников в `web/` повторите шаг 2, чтобы синхронизировать артефакты.
- Закоммитьте изменения и дождитесь, пока GitHub перечитает папку `docs`.

## Локальная проверка

```bash
python3 -m http.server --directory docs 8000
# Откройте http://127.0.0.1:8000
```

Такой подход имитирует GitHub Pages и помогает проверить, что все относительные пути и ресурсы подключаются корректно.
